# Provider 安全邊界

這份文件是多 AI 協作調度最重要的紅線。

多模型分工不是只有能力問題，也有資料流向問題。

## 一、先分兩種資料

### 可外送的資料

- 公開網頁
- 新聞
- 產品頁
- 公開 API 文件
- 不涉及你隱私與客戶資訊的素材

### 不可外送的資料

- 你的知識庫
- 你的客戶資料
- 你的未公開文件
- 你的技能包內容
- 任何你不確定能不能被拿去訓練的內容

## 二、免費模型的定位

免費模型很適合做：
- 搜尋
- 公開資料整理
- 跑腿
- 初步彙整

但免費不代表沒有代價。
代價常常是資料留存、訓練使用權、供應商不可控。

所以原則不是「免費模型不好」，
而是「免費模型只碰你能接受外流的資料」。

## 三、常見安全分工

### 安全拆法

- Claude / ChatGPT 類可信工具：處理你自己的資料
- 其他外部 provider：處理公開搜尋與外部資料

### 危險拆法

- 把整份知識庫或技能包直接丟給資料政策不明的 provider
- 為了省額度，把客戶資料送進免費模型

## 四、在這個技能包裡怎麼執行

如果資料是敏感的：
- 只做第一層分工判斷
- 或只用可信 provider

如果資料是公開的：
- 可以用任何可接受的 provider 做搜尋和蒐集

如果你不確定：
- 先當成敏感資料處理
- 不要因為方便就冒險

## 五、Provider 選擇原則

### Claude / Cowork / Claude Code

適合：
- 你的知識庫
- 需要長脈絡的整合
- 最終判斷

### Codex / GPT 類

適合：
- 單檔外部視角
- 局部改寫
- 交叉驗證

## 六、最終原則

多 AI 協作的真正價值，不值得用資料安全去交換。

如果一個 provider 很強，但你不確定資料會怎麼被使用，那它就不該碰你的核心資料。
