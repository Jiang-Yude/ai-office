## 核心概念

多 AI 協作不只是「跨品牌分工」（Claude + Codex + 其他家），也可以是「同品牌分級」（高／中／低）。兩種都是讓對的任務配對對的模型，降低成本、提升效率。

---

## 三個等級：低中高

統一用低中高描述，跨品牌通用。

| 等級 | 特性 | Claude | OpenAI |
|------|------|--------|--------|
| 高 | 判斷、創作、意圖解讀 | Opus | o1 / o3 |
| 中 | 整合、操作、規則比對 | Sonnet | GPT-4o |
| 低 | 執行、腳本、格式化 | Haiku | GPT-4o mini |

Claude 的參考定價（2026-03）：

| 等級 | 模型 | 輸入／百萬 token | 輸出／百萬 token |
|------|------|-----------------|-----------------|
| 高 | Opus 4.5 | $5 | $25 |
| 中 | Sonnet 4.6 | $3 | $15 |
| 低 | Haiku 4.5 | $1 | $5 |

高的輸出成本是中的 1.67 倍、低的 5 倍。

## 通用判斷標準

### 需要高的情境

- 有語氣、口語感要求的寫作（脆文、方格子）
- 模糊口述的意圖解讀
- 複雜的跨步驟判斷與策略規劃
- 需要創作能力的內容生成

### 中就夠的情境

- 讀檔、格式操作、看板開卡
- 規則比對、清單檢查
- 資訊整合、搜尋結果摘要
- 套模板產出（HTML 圖卡、Markdown 格式）
- 結構固定的文件寫入

### 低就夠的情境

- 純執行腳本（Playwright 截圖、Bash 指令）
- 簡單讀寫操作
- 時間戳、檔名格式化

## 脆文工作流程的分級範例

以「寫脆文 + 做圖卡 + 上看板」為例：

| 步驟 | 工作內容 | 等級 |
|------|----------|------|
| 1 | 讀技能包規則、讀看板現況 | 中 |
| 2 | 寫脆文初稿 | 高 |
| 3 | 搜尋資料驗證事實 | 中 |
| 4 | 理解模糊口述、改結構 | 高 |
| 5 | 品質檢查（規則比對） | 中 |
| 6 | 做圖文卡片 HTML | 中 |
| 7 | Playwright 截圖 | 低 |
| 8 | 開卡到看板 | 中 |

真正需要高的步驟：步驟 2、4，約佔 20-30% 工作量。

## 技術實作（Claude Code）

Claude Code 的 Agent 工具有 `model` 參數，可以指定子代理用哪個模型：

```
主 session：高（指揮官，負責判斷調度）
子代理：Agent 工具 + model: 中（執行機械性任務）
子代理：Agent 工具 + model: 低（跑腳本）
```

Cowork 目前不支援切換模型，全程跑同一個模型。

---

## 延伸方向

高中低的分級也可以跨品牌混搭：

- 高：Claude Opus（寫作、意圖判斷）
- 中：Sonnet 或 GPT-4o（搜尋整合、規則比對）
- 低：Codex / OpenCode / Haiku（腳本執行、批次操作）
