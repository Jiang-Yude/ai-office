# Claude Code 總控版｜多 AI 交叉分析執行流程

> 此文件僅適用於 Claude Code 作為總控台的場景。
> Codex 有自己的總控版流程，不在此文件範圍。

## 定位

Claude Code 同時扮演兩個角色：
1. **參賽者**：自己也做分析（跟 Codex 同等級）
2. **總控台**：負責調度 Codex、建檔、收斂、產出最終結果

## 環境前提

| AI | 呼叫方式 | 費用來源 | 自動化程度 |
|---|---|---|---|
| Claude（自己） | 直接執行 | Claude Code 額度 | 全自動 |
| Codex (GPT) | 使用者開另一個 Codex 視窗 | ChatGPT Pro 訂閱 | 半自動（需使用者觸發） |

## 檔案架構

所有產出集中在一份結果檔 + 一份 Codex 指令檔。

結果檔命名：`YYYY-MM-DD-HHMM 主題 兩家AI交叉分析.md`
Codex 指令檔命名：`YYYY-MM-DD-HHMM 給Codex的指令（第X輪）.md`

結果檔結構（heavy 模式）：
```markdown
## 第一輪：各自獨立分析
### Claude 分析
### Codex (GPT) 分析

## 第二輪：交叉評析
### Claude 評另一家分析
### Codex (GPT) 評另一家分析

## 最終總結
```

結果檔結構（light 模式）：
```markdown
## 第一輪：各自獨立分析
### Claude 分析
### Codex (GPT) 分析

## 最終總結
```

## 執行流程

### 第一輪：各自獨立分析

兩路同時出發：

**Claude（直接做）：**
1. 讀取原始素材檔
2. 以指定角色（如產品策略分析師）做獨立分析
3. 寫入結果檔的「Claude 分析」區段

**Codex（產指令檔）：**
1. 建立指令檔，內容包含：
   - 讀哪份原始素材
   - 分析角色與要求
   - 寫完後更新哪份結果檔的哪個區段
2. 告知使用者去 Codex 視窗執行：「讀取 `給Codex的指令（第一輪）.md` 然後照做」

**等待機制：**
- Claude 通常先完成
- Codex 需要使用者觸發，使用者回報「Codex 寫好了」後才進第二輪
- Claude Code 可以先用 `Read` 確認結果檔中 Codex 區段是否已被填入

### 第二輪：交叉評析（僅 heavy 模式）

確認兩家第一輪分析都到位後，兩路再次同時出發：

**Claude（直接做）：**
1. 讀取結果檔中兩家的第一輪分析
2. 分析兩家的異同、各自盲點、互補之處
3. 寫入「Claude 評另一家分析」區段

**Codex（產指令檔）：**
1. 建立第二輪指令檔
2. 告知使用者去 Codex 執行

### 最終總結

所有輪次完成後，Claude Code 負責收斂：

1. 讀取所有分析（light：兩份；heavy：四份）
2. 整理共識點、分歧點、各家獨特觀點
3. 給出綜合建議
4. 寫入結果檔的「最終總結」區段
5. 如果使用者要求，產出 Obsidian Canvas（.canvas）

### Canvas 產出規則

使用者說「做成畫布」時，產出 `.canvas` 檔：
- 每個分析是一個節點（嵌入對應段落）
- 第一輪兩個節點排成一排
- 第二輪兩個節點排在下一排（heavy 才有）
- 最終總結在最下方
- 節點之間用箭頭連線表示資料流向

## 給 Codex 的指令檔模板

### 第一輪指令檔

```markdown
## 指令

請你做以下事情：

1. 讀取這份檔案：`{原始素材檔名}`
2. 以{角色}的角度，做一份完整的獨立分析，包含：
   {具體要求清單}
3. 分析要具體、有觀點，不要只列優缺點，要給明確建議
4. 寫完後，打開 `{結果檔名}`，找到「### Codex (GPT) 分析」，把「（等待 Codex 寫入...）」替換成你的分析內容
5. 用繁體中文
```

### 第二輪指令檔

```markdown
## 指令

請你做以下事情：

1. 讀取 `{結果檔名}`
2. 找到「第一輪：各自獨立分析」下的兩份分析（Claude、Codex）
3. 分析這兩份分析的：
   - 共識點：兩家都同意的判斷
   - 分歧點：各家不同的判斷，以及你認為誰更合理
   - 盲點：兩家都漏掉的面向
   - 綜合建議
4. 寫完後，找到「### Codex (GPT) 評另一家分析」，把佔位文字替換成你的分析
5. 用繁體中文
```

## 容錯處理

| 狀況 | 處理方式 |
|---|---|
| Codex 超過 10 分鐘沒寫入 | 提醒使用者檢查 Codex 狀態 |
| 結果檔被其他程序鎖定 | 等 3 秒重試，三次失敗後告知使用者 |
| 素材檔過長 | 自動摘要後餵入，並在結果檔註明「讀的是摘要版」 |

## 注意事項

- Claude Code 是總控台，但作為參賽者時要假裝不知道另一家的分析（第一輪）
- 第一輪兩家的 prompt 要保持一致的角色設定和分析要求，確保可比性
- 不要在 Codex 指令檔裡放知識庫路徑以外的敏感資訊
- 每輪完成後主動告知使用者進度，不要悶頭做
