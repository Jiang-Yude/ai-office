# GPT 總控版｜多 AI 交叉分析執行流程

> 此文件僅適用於 GPT / Codex 作為總控台的場景。
> Claude Code 有自己的總控版流程，請讀 `claude-code-controller.md`。

## 定位

Codex 同時扮演兩個角色：
1. **參賽者**：自己也做分析，作為 GPT 這一路的代表
2. **總控台**：負責調度 Claude CLI、建檔、收斂、產出最終結果

這份文件的核心前提是：
- 使用者只跟 Codex 說話
- Codex 代為呼叫 Claude
- 所有結果最後回收成 Obsidian 檔案與畫布

## 環境前提

| AI | 呼叫方式 | 費用來源 | 自動化程度 |
|---|---|---|---|
| GPT（自己） | Codex / Codex CLI | OpenAI | 全自動 |
| Claude | Claude CLI | Claude Code / CLI 額度 | 全自動 |

必要條件：
- `codex` 可執行
- `claude` 可執行
- 知識庫可讀寫

## 角色分工

### GPT（Codex）

主責：
- 啟動任務
- 寫第一輪自己的分析
- 在 heavy 模式下寫第二輪「評另一家分析」
- 做最後總結
- 建立 `.canvas`

### Claude

主責：
- 第一輪獨立分析
- heavy 模式下第二輪交叉評析
- 補強 GPT 沒有看到的觀點

## 檔案架構

結果集中在：
- 一份最終結果檔
- 一個材料包資料夾
- 一張畫布（若使用者要求）

建議命名：
- 結果檔：`YYYY-MM-DD-HHMM 交叉比對：主題.md`
- 材料包：`_cross-check/YYYY-MM-DD-HHMM-主題/`
- 畫布：`YYYY-MM-DD-HHMM 兩家交叉分析畫布：主題.canvas`

## 兩種強度

### 輕度交叉分析 `light`

流程：
1. GPT 做第一輪獨立分析
2. Claude 做第一輪獨立分析
3. GPT 讀兩份結果，寫最終總結

適合：
- 一般企劃初審
- 文章草稿
- 想快速看兩家有沒有共識

### 重度交叉分析 `heavy`

流程：
1. GPT 做第一輪獨立分析
2. Claude 做第一輪獨立分析
3. GPT 評另一家第一輪分析的異同
4. Claude 評另一家第一輪分析的異同
5. GPT 讀四份材料，寫最終總結

適合：
- 重要決策
- 比賽方向選擇
- 高風險企劃
- 想保留「模型如何看彼此」的推理痕跡

## 執行流程

### 第一輪：各自獨立分析

兩路同時出發：

**GPT（自己）：**
1. 讀取原始素材
2. 依指定模式分析
3. 寫入第一輪 GPT 區段

**Claude（CLI 呼叫）：**
1. 把原始素材全文嵌入 prompt
2. 呼叫 `claude -p`
3. 回寫第一輪 Claude 區段

### 第二輪：交叉評析（僅 heavy）

確認兩家第一輪都完成後，兩路再次出發：

**GPT（自己）：**
1. 讀兩家第一輪分析
2. 分析共識、分歧、盲點
3. 寫入第二輪 GPT 區段

**Claude（CLI 呼叫）：**
1. 把兩家第一輪分析全文嵌入 prompt
2. 呼叫 `claude -p`
3. 回寫第二輪 Claude 區段

### 最終總結

GPT / Codex 負責收斂：

1. `light` 讀兩份
2. `heavy` 讀四份
3. 整理共識點、分歧點、獨見、行動建議
4. 寫入最終總結
5. 若使用者要求，建立 `.canvas`

## 畫布規則

使用者說「做成畫布」時：
- 原始素材在最左
- 第一輪兩家分析排成第一排
- heavy 時第二輪兩家分析排成第二排
- 最終總結放在最右或最下
- 箭頭表示資料流向，不表示因果正確性

## Prompt 原則

### 第一輪 prompt

要求兩家一致：
- 同一份原始素材
- 同一種模式（outside / style / blindspot）
- 同一種輸出語言（繁體中文）
- 同一組分析要求

### 第二輪 prompt

要求兩家一致：
- 明確比較另一家第一輪分析
- 區分共識與分歧
- 指出誰的判斷最值得優先採納
- 指出仍缺什麼資訊

## 容錯處理

| 狀況 | 處理方式 |
|---|---|
| Claude CLI 不可用 | 跳過 Claude，標記缺席，僅產出 GPT 單家版本 |
| 某家第二輪失敗 | 保留已完成材料，GPT 照樣總結 |
| 材料過長 | 先摘要再交叉分析，並在結果檔註明 |
