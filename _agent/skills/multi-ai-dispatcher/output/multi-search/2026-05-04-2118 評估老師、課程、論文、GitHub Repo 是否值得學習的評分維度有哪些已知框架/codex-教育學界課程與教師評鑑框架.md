## 關鍵重點
- 老師評鑑常見維度：教學規劃、學習環境、學習經驗／教學互動、專業責任。（來源 1：https://danielsongroup.org/the-framework-for-teaching/）
- 線上／混成課程設計常見維度：課程導覽、學習目標、評量、教材、互動活動、科技、學習支持、可近用性。（來源 2：https://www.qualitymatters.org/qa-resources/rubric-standards/higher-ed-rubric）
- 包容性課程設計常見維度：Engagement、Representation、Action & Expression。（來源 3：https://udlguidelines.cast.org/?azure-portal=true）
- 教師觀課評鑑常見維度：標準本位規劃、標準本位教學、學習條件、專業責任。（來源 4：https://marzanoevaluationcenter.com/evaluation/teacher-evaluation-model/）
- 教室互動品質常見維度：emotional support、classroom organization、instructional support；並以 1–7 分量表觀察。（來源 5：https://www.worldbank.org/en/programs/sief-trust-fund/brief/the-classroom-assessment-scoring-system-class）
- 課程／教育方案評鑑常見維度：Context、Input、Process、Product，可用來看需求、資源、執行、成果。（來源 6：https://www.betterevaluation.org/fr/node/1261）
- 訓練成效評估常見維度：Reaction、Learning、Behavior、Results，偏向學習後是否轉化為行為與成果。（來源 7：https://www.cdc.gov/training-development/php/qts/quality-training-standards-references.html）

## 不確定點
- 從「教育學界課程與教師評鑑框架」角度，找到的是老師、課程、教學設計、訓練成效框架；沒有找到直接評估「GitHub Repo 是否值得學習」的教育學界標準框架。
- 論文是否值得學習，在教育學界通常會轉向「研究品質評讀／證據品質」框架，不屬於本次角度的主要教師與課程評鑑框架。
