## 關鍵重點
- Karpathy 的原始 gist 把 LLM Wiki 定義為「用 LLM 建立個人知識庫」的模式，這是後續 GitHub 專案的核心引用來源。（來源 1：https://gist.github.com/karpathy?direction=desc&sort=updated）
- GitHub topic `llm-wiki` 已出現 75 個公開 repositories，代表這個模式已形成一批開源實作，而不只是單一專案。（來源 2：https://github.com/topics/llm-wiki）
- 正面評價的共同賣點是「LLM 維護 wiki / markdown 知識庫」，把 raw sources 轉成持續累積的結構化知識。（來源 3：https://llmwiki.app/）
- `nashsu/llm_wiki` 顯示 1.7k stars、185 forks，且有 2026-04-12 release，屬於目前較有 GitHub 熱度的 LLM Wiki 實作。（來源 4：https://github.com/nashsu/llm_wiki）
- `NiharShrotri/llm-wiki` 強調 local、Ollama、Qwen3、QMD、Obsidian，正面訊號集中在「本地化」「混合搜尋」「可寫回 wiki」。（來源 5：https://github.com/NiharShrotri/llm-wiki）
- `Pratiyush/llm-wiki` 的正面定位是把 Claude Code、Codex CLI、Copilot、Cursor、Gemini CLI session 轉成 searchable、interlinked knowledge base。（來源 6：https://github.com/Pratiyush/llm-wiki）
- Denser.ai 的文章把此模式描述為兩週內快速擴散，並提到 stars、forks、comments、GitHub re-implementations，屬於外部正面採用訊號。（來源 7：https://denser.ai/blog/llm-wiki-karpathy-knowledge-base/）

## 不確定點
- 搜尋主題包含 `2025`，但我找到最具代表性的 LLM Wiki / Karpathy pattern 資料多集中在 2026 年 4 月；未找到明確標示為 2025 GitHub trending 的同一波資料。
- `GitHub trending` 的即時榜單本身會變動；上述資料主要來自 GitHub topic、repository 頁面與近期文章，不等於 GitHub 官方 Trending 歷史榜單。
- 部分 GitHub 頁面只顯示 updated、release 或 crawled 時間，沒有完整發文日期。
