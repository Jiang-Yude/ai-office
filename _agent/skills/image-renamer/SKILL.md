# 圖片看圖命名技能包

這個技能包的核心概念很簡單：圖片的名稱應該反映它的內容，而不是截圖時間戳或隨機字串。
透過逐一查看每張圖片，產出讓人一眼就能理解的中文名稱。

## 工作流程

### 第一步：確認來源與目標

先問清楚（如果使用者沒說的話）：
- 來源資料夾在哪？
- 要輸出到哪裡？（新資料夾？還是直接在原地改名？）

如果使用者已經說清楚了，直接開始，不用再問。

### 第二步：列出所有圖片

用 `ls` 列出資料夾裡的所有圖片檔案（.png、.jpg、.jpeg、.webp、.gif 等）。

### 第三步：逐一看圖、產出中文名稱

用 Read 工具逐一讀取每張圖片，觀察內容，然後為每張圖命名。

命名原則：
- 格式：`主題-細節描述.副檔名`（用 `-` 串連，不用空格）
- 用繁體中文
- 名稱要能讓人不看圖就大概猜到內容（像是「Codex操作介面-建立專案提示詞視窗」、「AI總管工作流程-三工具整合圖」）
- 避免太籠統（例如「截圖1」「未命名」「圖片」）
- 如果圖片裡有明確的標題或文字，可以直接引用作為命名依據
- 圖片中出現的英文品牌名稱（如 Codex、Antigravity、Cowork、Claude、Obsidian）要拼寫正確，從圖片內容中確認，不要憑印象拼

### 第四步：處理時間戳記

看原始檔名裡有沒有時間資訊：

情況一：原始檔名包含時間（例如 `截圖 2026-03-14 下午4.54.24.png`）
→ 從中抽取日期和時間，轉成 `YYYY-MM-DD HHMM` 格式，加在中文名稱前面
→ 例如：`2026-03-14 1654_工作日誌範本-事件內容分層整理法.png`

情況二：原始檔名沒有時間資訊（例如 `unnamed.png`、`image.webp`、或一串隨機數字）
→ 直接用中文名稱，不加時間前綴
→ 例如：`AI數位實戰-本週末兩場免費線上講座海報.png`

時間轉換規則（macOS 截圖格式）：
- `下午1.00` → `1300`
- `下午4.54` → `1654`
- `下午6.20` → `1820`
- `晚上7.22` → `1922`
- `上午9.30` → `0930`

### 第五步：輸出

如果使用者要新資料夾：
- 建立目標資料夾
- 用 `cp` 複製並同時重新命名（保留原始檔案）

如果使用者要直接改名：
- 用 `mv` 在原資料夾裡重新命名

完成後，列出對照表給使用者確認：

```
原始名稱 → 新名稱
截圖 2026-03-14 下午4.54.24.png → 2026-03-14 1654_工作日誌範本-事件內容分層整理法.png
unnamed.png → AI數位實戰-本週末兩場免費線上講座海報.png
...
```

## 注意事項

- 遇到非圖片檔案（.md、.json、資料夾等）跳過，不處理
- 如果一次有很多圖（超過 20 張），可以分批處理，先做前半再問使用者是否繼續
- 命名完成後，原始資料夾的檔案保持不動（除非使用者明確說要直接改名）
